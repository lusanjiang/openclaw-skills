# 系统监控与优化技能

## 概述

本技能负责监控服务器资源使用情况，自动检测并优化内存占用，确保系统稳定运行。

## 核心原则

> **不轻言放弃，持续尝试到底**
> 
> 除非用户主动要求放弃，否则不轻易放弃任务。遇到问题时：
> 1. 分析问题原因
> 2. 尝试多种解决方案
> 3. 持续迭代直到成功
> 4. 记录问题和解决方案

> **杜绝 AI 幻觉，保证回复真实性**
> 
> 所有回复必须基于真实数据、实际执行结果或可靠来源：
> - 不编造不存在的数据或文件内容
> - 不确定时明确说明"无法确认"或"需要验证"
> - 执行命令后如实汇报结果，不夸大或隐瞒
> - 引用外部资料时注明来源，不虚构引用

## 监控指标

### 内存监控
- **警告阈值**: 80%
- **危险阈值**: 90%
- **自动清理阈值**: 85%

### CPU 监控
- **警告阈值**: 80%
- **危险阈值**: 95%

### 磁盘监控
- **警告阈值**: 80%
- **危险阈值**: 90%

## 自动优化策略

### 1. 浏览器进程管理

**检测**: Chrome/Chromium 进程内存占用

**优化动作**:
```bash
# 当浏览器进程占用超过 2GB 时
pkill -f chrome
```

**预防**:
- 每次浏览器使用后强制关闭
- 限制同时运行的浏览器实例数为 1
- 设置浏览器超时自动关闭（5分钟无操作）

### 2. 临时文件清理

**清理目标**:
```bash
# Playwright 临时文件
rm -rf /tmp/playwright-download-*

# Node 编译缓存
rm -rf /tmp/node-compile-cache/*

# 系统临时文件
find /tmp -type f -atime +1 -delete
```

### 3. 日志文件管理

**清理策略**:
- 日志文件超过 100MB 时轮转
- 保留最近 7 天的日志
- 压缩历史日志

## 手动检查命令

### 检查内存使用
```bash
free -h
ps aux --sort=-%mem | head -10
```

### 检查浏览器进程
```bash
ps aux | grep chrome | grep -v grep
```

### 清理浏览器进程
```bash
pkill -f chrome
```

## 定时任务

**内存监控任务**:
- 频率: 每 30 分钟
- 动作: 检查内存使用，超过阈值自动清理

**磁盘清理任务**:
- 频率: 每天凌晨 3 点
- 动作: 清理临时文件和旧日志

## 内存优化建议

### 浏览器使用
1. **及时关闭** - 使用完毕后立即关闭浏览器
2. **单实例** - 同时只运行一个浏览器实例
3. **超时关闭** - 设置 5 分钟无操作自动关闭
4. **清理数据** - 定期清理浏览器缓存数据

### Python 进程
1. **及时释放** - 大对象使用后立即删除
2. **限制并发** - 避免同时运行多个内存密集型任务
3. **使用生成器** - 大数据处理使用生成器而非列表

### 系统配置
1. **增加 Swap** - 建议添加 2-4GB Swap 分区
2. **限制进程** - 使用 ulimit 限制单进程内存
3. **监控告警** - 设置内存告警阈值

## 常见问题

### Q: 浏览器进程无法关闭
**解决**: 使用 `pkill -9 -f chrome` 强制关闭

### Q: 内存清理后仍然不足
**解决**: 
1. 重启 OpenClaw 服务
2. 检查是否有僵尸进程
3. 考虑升级服务器配置

### Q: 定时任务未执行
**检查**:
```bash
openclaw cron list
openclaw cron status
```
