## 🏗️ 工作流架构 (v3.1 新特性)

学习自 [Coze 扣子](https://www.coze.cn) 的工作流设计理念，实现可视化编排：

```
定时触发(14:30) → 获取基金数据 → 获取沪深300 → 计算收益率
                                                        ↓
发送通知 ← 生成交易建议 ← LLM推理 ← 加载AI产业链数据
                                                        ↓
飞书同步 ← 保存建议 ← 检测紧急预警
```

### 节点类型
| 节点 | 功能 | 示例 |
|:---|:---|:---|
| `api_call` | 获取基金/市场数据 | 天天基金网、新浪财经 |
| `code` | 计算收益率 | Python 计算持仓收益 |
| `llm_reasoning` | 生成交易建议 | Kimi 分析并给出操作建议 |
| `selector` | 紧急预警检测 | 波动>±3%触发预警 |
| `database_write` | 同步飞书 | 写入多维表格 |
| `notify` | 发送用户通知 | 推送交易建议 |

### 配置文件
- `metadata.json` - Skill 元数据（轻量加载）
- `workflow.yaml` - 工作流定义（可视化编排）
- `SKILL.md` - 详细文档（按需加载）

## ⚠️ 核心约束

### 交易时间窗口
| 时间节点 | 含义 | 操作要求 |
|:---|:---|:---|
| **14:30** | 提醒时间点 | 系统必须在此之前生成建议 |
| **14:50** | 最后决策时间 | 给出明确买入/卖出/观望指令 |
| **15:00** | 收盘截止线 | 15:00前操作=当日净值，之后=次日净值 |

### T+1交易规则
```
T日15:00前买入 → T日收盘净值成交 → T+1日确认份额 → T+2日才能卖出

举例:
- 周一14:50买入 → 按周一净值 → 周二确认 → 周三可卖
- 周五减仓后 → 下周一资金可用（T+2实际可用）
```

- **夏普比率**：2.50（优于同类）

#### 2. 业绩分化明显

#### 3. 调仓时机
- **季度调仓**：每次季报都有明显持仓变化
- **追涨杀跌**：会在趋势确认后加仓，如2023年追AI

## 📊 交易策略

### 止盈策略（"三二一法则"）
| 轮次 | 目标收益率 | 操作 |
|:---|:---|:---|
| 第一次 | 30% | 止盈30%，剩余继续持有 |
| 第二次 | 20% | 止盈20-30% |
| 第三次 | 10% | 全部止盈，等待下一轮 |

### 止损策略
- **绝对止损**: -15%
- **动态止损**: 盈利后回撤5-8%立即止盈

### 买入策略
- **金字塔买入**: 30%→30%→40% 分批建仓
- **定投**: 每月固定金额，长期持有

### 加仓信号
- 大盘当日下跌 ≥ 2%
- 连续3天下跌，第4天继续跌
- 个人目标仓位未满，且大盘估值合理

### 卖出信号
- 大盘当日大涨 ≥ 2%，考虑止盈部分
- 周五开盘减仓至50%（周末效应）

## 🔗 飞书多维表格共享知识库

### 方案介绍
使用飞书多维表格作为AI助手之间的共享数据库，实现：
- 实时数据同步
- 多AI协作编辑
- 历史记录追溯
- 可视化展示

### 表格结构

| 表名 | 用途 | 字段 |
|:---|:---|:---|
| **交易记录** | 记录每日操作 | 日期、操作、金额、收益率、AI建议 |
| **持仓监控** | 跟踪持仓状态 | 仓位、收益、预警状态、目标位 |
| **市场数据** | 存储市场信息 | 沪深300、基金估算、市场环境 |
| **AI知识库** | 知识沉淀 | 类别、标题、内容、来源、重要性 |

### 配置步骤

#### 1. 创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 添加权限: bitable:app, bitable:record, bitable:field
4. 发布应用并获取App ID/Secret

#### 3. 配置API代码
编辑 `/root/.openclaw/workspace/feishu_bitable_api.py`:
