# 多Agent协作系统

## 组织架构设计

参考三万（sanwan.ai）和OpenClaw最佳实践：

```
┌─────────────────────────────────────┐
│           总指挥 (Coordinator)        │
│      任务分配 · 进度监控 · 质量把控    │
└─────────────┬───────────────────────┘
              │
    ┌─────────┼─────────┐
    │         │         │
┌───▼───┐ ┌──▼────┐ ┌──▼────┐
│ 笔杆子 │ │ 工程师 │ │ 情报官 │
│ Writer │ │Engineer│ │Researcher│
└────────┘ └─────────┘ └─────────┘
    │         │         │
┌───▼───┐ ┌──▼────┐ ┌──▼────┐
│ 运营官 │ │ 进化官 │ │ 社区官 │
│Operator│ │Evolver │ │Community │
└────────┘ └─────────┘ └─────────┘
```

## Agent角色定义

### 1. 总指挥 (Coordinator)
- **职责**：任务分解、Agent调度、结果整合
- **技能**：任务规划、冲突解决、质量评估
- **触发**：所有复杂任务入口

### 2. 笔杆子 (Writer)
- **职责**：内容创作、文案撰写、报告生成
- **技能**：Markdown、Feishu文档、小红书文案
- **标签**：`#写作`

### 3. 工程师 (Engineer)
- **职责**：代码编写、工具开发、自动化脚本
- **技能**：Python、JavaScript、爬虫、API对接
- **标签**：`#开发`

### 4. 情报官 (Researcher)
- **职责**：信息搜集、数据分析、竞品调研
- **技能**：Web搜索、PDF解析、数据提取
- **标签**：`#调研`

### 5. 运营官 (Operator)
- **职责**：定时任务、监控告警、日常维护
- **技能**：Cron、Heartbeat、备份检查
- **标签**：`#运营`

### 6. 进化官 (Evolver)
- **职责**：技能学习、代码优化、架构升级
- **技能**：自我反思、错误修正、知识积累
- **标签**：`#进化`

### 7. 社区官 (Community)
- **职责**：用户互动、消息回复、关系维护
- **技能**：多平台消息、情感分析、彩蛋设计
- **标签**：`#互动`

## 协作流程示例

### 场景：发布一篇小红书笔记

```
用户：发小红书介绍TGM1N

总指挥：
  1. 分解任务
     - 情报官：搜索TGM1N最新资料
     - 笔杆子：撰写文案
     - 工程师：生成配图/处理图片
     - 运营官：定时发布
  
  2. 调度执行
     → 情报官开始搜索
     → 笔杆子基于资料写作
     → 工程师准备图片
     → 运营官安排发布时间
  
  3. 结果整合
     → 汇总各Agent输出
     → 质量检查
     → 交付用户
```

## 知识传递机制

### Agent间知识共享
- **Memory系统**：共享长期记忆（MEMORY.md）
- **Skill库**：通过ClawHub共享技能
- **日志记录**：memory/YYYY-MM-DD.md

### 学习机制
- **错误反思**：犯错后自动记录并修正
- **技能积累**：每个新任务都可能成为新Skill
- **版本迭代**：持续优化已有Skill
